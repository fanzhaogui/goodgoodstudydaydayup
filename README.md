    目的是建立对象间的关联；

    它有两个原则：
    1. 按接口编程而不是按实现编程
    2. 优先使用组合而不是继承

[TCP/IP 协议](https://www.cnblogs.com/onepixel/p/7092302.html)

> TCP/IP协议栈是一系列网络协议的总和。

> 它定义了电子设备如何连入因特网，以及数据如何在他们之间进行传输。

> TCP/IP 协议采用4层结构，分别是

    应用层、传输成。网路层、链路层
    
> 每一层都呼叫它的下一层所提供的协议来完成之间的需求。

> 由于我们大部分时间都工作在应用层，下层的事情不用我们操心；

> 其次网络协议体系本身就很复杂庞大，入门门槛高，因此很难搞清楚；

> 通俗一点讲就是“一个主机的数据要经过哪些过程才能发送到对方的主机上”
